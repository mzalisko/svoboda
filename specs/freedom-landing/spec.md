## 1. Загальна мета

Створити вебсайт-лендинг для книги **"Свобода"** на WordPress як **кастомну тему**, без використання CSS/JS-фреймворків, білдерів сторінок і плагінів-конструкторів.

- Чистий **HTML5**
- Чистий **CSS** (без Bootstrap, Tailwind, SASS-фреймворків)
- Чистий **PHP** за стандартною Template Hierarchy WordPress
- Без WooCommerce
- Продаж книги (паперова / електронна) через кастомну форму + кастомну бізнес-логіку

---

## 2. Контент і керування даними (ACF)

Весь змінний контент керується через **Advanced Custom Fields**:

- Окрема **ACF-група** для контенту головної сторінки
- **ACF Options Page** для глобальних налаштувань (контакти, соцмережі, тексти футера)
- **ACF Repeater / Flexible Content** для повторюваних блоків:
  - перелік больових точок аудиторії (блок "Ця книга для вас, якщо...")
  - шість пунктів трансформаційного шляху
  - картки відгуків (карусель)
  - три пункти блоку "Для кого" (власники бізнесу)

---

## 3. Замовлення та оплата (без WooCommerce)

### 3.1 Форма замовлення
- Рендер полів і базова валідація: **Contact Form 7** або **WPForms** (готовий плагін)
- Вся бізнес-логіка — кастомний код теми або **mu-plugin**
- Варіанти: паперова (450₴) / електронна (290₴)

### 3.2 Нова Пошта (для паперової версії)
- Вибір міста та відділення з автопідстановкою через AJAX
- Кастомний REST-ендпоінт WordPress: `register_rest_route`
- Ендпоінт проксує запити до публічного API Нової Пошти
- **Ключ API зберігається на сервері** (ACF Options Page або константа в `wp-config.php`) і **ніколи не потрапляє у фронтенд**

### 3.3 Оплата — Monobank Acquiring
- Сервер створює інвойс через **Merchant API** за токеном (зберігається на бекенді)
- Користувача перенаправляють на сторінку оплати Monobank
- Окремий **webhook-ендпоінт** приймає статус оплати й оновлює запис замовлення
- Замовлення зберігається як кастомний тип запису `order` (CPT) або в окремій таблиці БД

> ⚠️ **Безпека секретів**: токен Monobank і ключ API Нової Пошти ніколи не передаються у JS/фронтенд. Всі виклики — виключно server-side (REST-ендпоінти теми / mu-plugin).

## 4. Структура сторінки (one-page лендинг)

Секції йдуть послідовно, порядок фіксований і однаковий на десктопі й мобільному (без перестановки блоків):

1. **Hero** — заголовок "СВОБОДА", підзаголовок, ілюстрація паперового літачка, цитата
2. **Замовлення книги** — 2 варіанти (паперова 450₴ / електронна 290₴), позначка обмеженого тиражу, 3D-візуалізація обкладинки
3. **Про автора** — біографія, фото Ігоря Браславця, посилання на Instagram (`@braslavetsigor`)
4. **"Ця книга для вас, якщо ви відчуваєте"** — список болей з червоними ромбовидними маркерами, 3 фото з'єднані кривою лінією
5. **Відеоблок** — фото автора на природі, кнопка відтворення, підпис (тривалість, ім'я)
6. **Цитатний блок** — зі скетч-ілюстрацією
7. **Трансформація** — звивиста SVG-лінія-маршрут, 6 пунктів шляху, анімований літачок "приземляється" на текстуровану планету
8. **"Для власників бізнесу / Для кого"** — нумерація 01/02/03, фото, кнопка "Презентація"
9. **Карусель відгуків** — темний зоряний фон, стрілки навігації, крапки пагінації
10. **Фінальна CTA** — фонове фото людини на вершині гори, ті самі кнопки замовлення

### Брейкпоінти
- Мобільна версія: до 768px (референс 375px у Figma), суцільний вертикальний стек
- Десктоп: контент max-width ≈ 1200–1400px, бокові відступи; полотно дизайну — 1920px

## 5. Дизайн-система (точні значення з Figma-інспектора)

### 5.1 Кольори (CSS-змінні)
```css
:root {
  --color-accent-red: #F01212;           /* 90% opacity для заголовків/цитат */
  --color-accent-red-solid: rgba(240,18,18,0.9); /* 100% для кнопок/цін */
  --color-text-primary: #323232;
  --color-text-secondary: #606060;
  --color-bg-alt: #F5F5F5;
  --color-bg-base: #FFFFFF;
}
```

### 5.2 Типографіка
| Елемент | Шрифт | Накреслення | Line-height | Tracking | Колір |
|---|---|---|---|---|---|
| Великі заголовки (H1/H2, ~100–135px) | Cormorant Garamond | SemiBold | 128% (≈0.94em) | 2% | --color-accent-red |
| Основний текст / підзаголовки | Times New Roman | Regular | 110% | 2% | --color-text-primary |

- Cormorant Garamond: підключення через Google Fonts, **бажано self-hosted** (швидкість)
- Times New Roman: web-safe fallback / системний serif-стек: `"Times New Roman", Times, serif`

### 5.3 Кнопки-пігулки (order buttons)
- Фон: прозорий
- Обводка: ~2.5px, `--color-accent-red-solid`
- Border-radius: ~58px
- Padding: 30px (гор.) / 24px (верт.)
- Примітка: контур має легку рукописну нерівність → якщо потрібна повна візуальна точність, використати експортований SVG-асет замість чистого CSS border

## 6. Анімації (пріоритетний розділ)

Всі анімації нижче виявлені й точно зафіксовані з Figma (Smart Animate / Motion timelines), длительність базового циклу — **3000ms**, `animation-direction: alternate` (boomerang), де застосовно.

| Елемент | Тип анімації | Опис |
|---|---|---|
| Шапка сайту | Scroll listener | Стає компактнішою і напівпрозорою при скролі (клас додається JS через `scroll` event) |
| Заголовки Hero | Fade-in + зсув | Послідовна поява із затримкою між елементами |
| Паперовий літачок (Hero) | CSS keyframes | М'яка "плаваюча" анімація (translate + scale, ease-out → linear, infinite alternate) |
| Обкладинка книги | Vanilla JS | Нахил у 3D-перспективі залежно від позиції курсору |
| "Про автора" / "Ця книга для вас" | IntersectionObserver (threshold 0.2) | Поява по скролу: opacity + translate |
| Фото-колаж у блоці "для вас" (3 фото) | Keyframes: translate/scale | Легке "дихання"/зміщення в циклі 3s |
| Кнопка відтворення відео | Hover pulse + lazy load | Пульсація на hover; заміна прев'ю на вбудоване відео при кліку |
| SVG-лінія трансформації | scroll-linked stroke-dasharray/dashoffset | Промальовується поступово залежно від позиції прокрутки; пункти шляху підсвічуються синхронно |
| Куля/планета (кінець лінії) | opacity keyframe | Плавна поява (0 → 1) в кінці циклу |
| Літачок (трансформація) | mix-blend-multiply, translate | "Приземлення" на текстуровану планету |
| Карусель відгуків | Vanilla JS slider | Власна реалізація, підтримка свайпів дотиком, без Swiper/Slick |
| Фінальна CTA-секція | Паралакс | Фонове зображення |

### Приклад точного keyframe-набору (знято з Figma-інспектора, hero-цитата "❝"):
```css
.hero-quote-mark {
  animation: kf_quote_translate 3s linear, kf_quote_scale 3s linear;
  animation-iteration-count: infinite;
  animation-direction: alternate;
}
@keyframes kf_quote_translate {
  0%   { animation-timing-function: ease-out; translate: 60px 0px; }
  33.987% { translate: 0px 0px; }
  100% { translate: 0px 0px; }
}
@keyframes kf_quote_scale {
  0%   { animation-timing-function: ease-out; scale: 0.72 0.72; }
  33.987% { scale: 1 1; }
  100% { scale: 1 1; }
}
```

### SVG-лінія (path-trim через stroke-dasharray, вимагає `pathLength="1"` на SVG path):
```css
.transformation-line {
  animation: kf_line_trim 3s linear;
  animation-iteration-count: infinite;
  animation-direction: alternate;
}
@keyframes kf_line_trim {
  0%      { animation-timing-function: ease-out; stroke-dasharray: 0 1; stroke-dashoffset: -1; visibility: hidden; }
  50.304% { stroke-dasharray: 1 1; stroke-dashoffset: 0; visibility: visible; }
  100%    { stroke-dasharray: 1 1; stroke-dashoffset: 0; visibility: visible; }
}
```

> Повний набір знятих keyframe-даних (по кожному вузлу Figma) — див. `docs/figma/animations.md`.
> **Важливо**: у Figma-файлі це micro-loop анімації (для перегляду в самій Figma), а не буквальний скрол-тригер. Для продакшена їх треба переосмислити як:
> - infinite decorative loops (літачок, куля/фото-дихання) → залишити як CSS keyframes,
> - scroll-triggered появи (fade-in секцій, промальовка лінії) → реалізувати через `IntersectionObserver` / scroll-position логіку, використовуючи ці keyframes лише як референс для easing і тайминга, а не буквальний infinite loop.

## 7. Технічний стек і розгортання

### Docker Compose
- `db` — MySQL/MariaDB, том для даних, env для пароля/назви БД
- `wordpress` — офіційний образ `wordpress`, том з каталогом теми, env `WORDPRESS_DB_*`
- (опційно) `phpmyadmin` — для зручності адміністрування під час розробки

### Секрети
- Усі секрети (паролі БД, токен Monobank, ключ API Нової Пошти) — у `.env`, **не** в системі контролю версій
- `wp-config.php` з реальними обліковими даними — у `.gitignore`
