# Docker: як піднімати проекти (загальний гайд)

Універсальна шпаргалка для будь-якого проекту з `docker-compose.yml` — не лише для "Свободи".
Команди виконуються в терміналі (PowerShell/CMD/Terminal) з папки проекту, де лежить файл `docker-compose.yml`.

## 0. Що потрібно один раз

- Встановити **Docker Desktop** (Windows/Mac) — https://www.docker.com/products/docker-desktop/
- Переконатись, що Docker Desktop запущено (іконка кита в треї активна)
- Перевірити встановлення:
  ```
  docker --version
  docker compose version
  ```

## 1. Загальний алгоритм для будь-якого проекту

1. Відкрити термінал у папці проекту (там, де `docker-compose.yml`):
   ```
   cd шлях\до\проекту
   ```
2. Якщо є `.env.example` — скопіювати в `.env` і заповнити реальні значення:
   ```
   copy .env.example .env        (Windows CMD)
   Copy-Item .env.example .env   (PowerShell)
   ```
3. Підняти контейнери:
   ```
   docker compose up -d
   ```
   `-d` = у фоні (detached). Без цього прапорця термінал буде "зайнятий" логами.
4. Перевірити, що все запустилось:
   ```
   docker compose ps
   ```
   Статус усіх сервісів має бути `running` / `Up`.
5. Відкрити сайт у браузері за портом, вказаним у `docker-compose.yml` (розділ `ports:`, напр. `"8080:80"` → `http://localhost:8080`).

## 2. Команди на щодень

| Дія | Команда |
|---|---|
| Підняти всі сервіси у фоні | `docker compose up -d` |
| Зупинити (контейнери лишаються, дані зберігаються) | `docker compose stop` |
| Зупинити і видалити контейнери (том з даними лишається) | `docker compose down` |
| Зупинити і видалити **разом з даними** (обережно!) | `docker compose down -v` |
| Перезапустити один сервіс | `docker compose restart wordpress` |
| Подивитись логи всіх сервісів (наживо) | `docker compose logs -f` |
| Логи одного сервісу | `docker compose logs -f wordpress` |
| Список запущених контейнерів проекту | `docker compose ps` |
| Зайти в термінал контейнера | `docker compose exec wordpress bash` |
| Перезібрати образи після зміни Dockerfile | `docker compose up -d --build` |
| Прибрати "сирітські" контейнери зі старих версій compose-файлу | `docker compose up -d --remove-orphans` |

## 3. Типові проблеми

- **"port is already allocated"** — порт зайнятий іншим процесом/проектом. Або зупинити той процес, або змінити порт у `docker-compose.yml` (наприклад `"8090:80"` замість `"8080:80"`).
- **Сайт відкривається, але білий екран / помилка БД** — контейнер БД ще не встиг ініціалізуватись при першому запуску. Почекати 10-20 секунд і оновити сторінку, або перевірити `docker compose logs db`.
- **Зміни в коді теми/плагіна не видно на сайті** — перевірити, що папка змонтована як volume (`volumes:` у `docker-compose.yml`), а не скопійована всередину образу один раз.
- **Забув пароль/налаштування** — усі значення беруться з `.env`, він не в git. Якщо загубили — просто прописати нові значення (для нового оточення це не проблема, для існуючих даних БД — може знадобитись оновити пароль користувача БД теж).

## 4. Конкретно для цього проекту ("Свобода")

Сервіси в `docker-compose.yml`: `db` (MariaDB), `wordpress`, `phpmyadmin`.

```
cd C:\Dev\freeproject\my-project
copy .env.example .env
```

Відкрити `.env` і заповнити:
```
DB_NAME=svoboda
DB_USER=svoboda
DB_PASSWORD=будь-який-надійний-пароль
DB_ROOT_PASSWORD=інший-надійний-пароль
MONOBANK_TOKEN=            (можна лишити порожнім поки не тестуєте оплату)
NOVAPOSHTA_API_KEY=        (можна лишити порожнім поки не тестуєте доставку)
```

Підняти:
```
docker compose up -d
```

Перевірити статус:
```
docker compose ps
```

Відкрити в браузері:
- Сайт / установка WordPress: **http://localhost:8080**
- phpMyAdmin (керування БД): **http://localhost:8081** (сервер `db`, логін/пароль з `.env`)

При першому відкритті `http://localhost:8080` WordPress покаже майстер встановлення — обрати мову, задати назву сайту, логін/пароль адміністратора.

Після встановлення:
1. Увійти в адмінку: `http://localhost:8080/wp-admin`
2. Встановити плагін **Advanced Custom Fields** (потрібен для полів теми)
3. Вигляд → Теми → активувати тему **"Свобода"**
4. Налаштування → Читання → обрати статичну головну сторінку з шаблоном `front-page.php` (або створити сторінку і призначити її головною)

Зупинити, коли закінчили роботу:
```
docker compose stop
```
(дані БД і завантажені файли лишаються; наступного разу просто `docker compose up -d` знову).
