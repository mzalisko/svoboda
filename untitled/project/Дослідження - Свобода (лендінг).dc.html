«Свобода» — лендінг книги

Повний технічний розбір макета для точного відтворення вигляду, розмірів, відступів і анімацій.

Що це: односторінковий promo-сайт (landing page) для книги «Свобода» автора Ігоря Браславця — блоки: hero, замовлення книги, про автора, «для кого ця книга», відео-презентація, цитата, трансформація (roadmap), лід-магніт (email), для власників бізнесу, відгуки, повторний CTA.

Джерело правди: Page-1 → Desktop - 4 (вузол 180:62, 1920×12818) — фінальна десктопна версія, і Page-1 → mobile — фінальна мобільна версія. Desktop - 5 — майже ідентичний дубль (варіант/бекап, різниця лише в 1–2 картках відгуків — Frame164 замість Frame160). Page-2 і Page-3 — це чорнові ранні ітерації того ж лендінгу (інша палітра rgb(242,247,253), інший шрифт Manrope, інша структура) — вони НЕ фінальний дизайн, ігноруйте їх для відтворення; лишені тут для довідки про еволюцію макету.

1. Полотно і сітка

  • Desktop canvas: 1920 × 12818px, overflow hidden, фон #FFFFFF.
  • Контент-контейнер більшості секцій: горизонтальний padding 160px з кожного боку (content width ≈ 1600px), за винятком hero (абсолютне позиціонування).
  • Header: висота area 34px, top 26px, side margin 41px (ширина 1839px), flex row, justify space-between, по центру логотип «Свобода» (Times New Roman 30px).
  • Mobile canvas: окремий артборд у /Page-1/mobile, той самий контент вертикально стековано, single-column.

7. Декоративні SVG-лінії ("рука пише")

Тонкі криві сполучні лінії (сірі, rgb(103,103,103) / rgba(96,96,96,0.36), ~1–4px stroke) — це не декор "з коробки", а власні кастомні SVG-path'и: Vector-11.svg (вертикальна хвиляста лінія біля "Про автора"), Vector-11-2/3.svg (велика хвиля навколо H2 "Ця книга для вас"), нова хвиляста траса в блоці "Ви отримаєте трансформацію" (карти зигзагом), товста хвиляста лінія Vector-11-3.svg вздовж всього блоку "Для власників бізнесу"/"Відгуки" (369×1834). Скопіювати ці SVG 1-в-1 з фіга (не перевинаходити геометрію) — вони містять точні криві Безьє даного макету.

8. Анімації

У файлі є 148 KEYFRAME + 66 KEYFRAME_TRACK + 28 ANIMATION_PRESET_INSTANCE вузлів на рівні файлу (Figma Smart-Animate/Video preset), але вони НЕ резолвляться в JSX-експорті (це property nodes, не layout) — конкретні easing/duration не читаються цим інструментарієм. Що можна вивести зі структури й типового патерну для такого лендінгу:

  • Desktop-4 vs Desktop-5 — два майже ідентичні кадри з мінімальною різницею в блоці відгуків → найімовірніше це before/after стан Smart Animate прототипу (кнопка/картка змінюється при кліку/hover), а не окрема сторінка.
  • Хвилясті лінії-з'єднувачі — типовий патерн "draw-on-scroll" (stroke-dashoffset reveal синхронно зі скролом) — рекомендовано відтворити як scroll-triggered SVG path draw, а не статичний штрих.
  • Кругла плашка "ОТРИМАТИ" (email-форма у колі) і paper-plane SVG поруч — типово анімуються появою/легким рухом (float) при в'їзді в viewport.
  • Рекомендація агенту-виконавцю: якщо потрібна точна крива анімації — відкрити оригінал у Figma (Smart Animate) вручну; цей інструментарій дає лише структуру, не timing.

9. Мобільна версія

Той самий контент, порядок секцій ідентичний, одноколонковий стек (див. знімок нижче). Кнопки/картки на всю ширину, hero-текст по центру, header складається в гамбургер-меню (іконка списку зверху). Розміри шрифтів пропорційно зменшені — читати точні значення з /Page-1/mobile/index.jsx per-елемент (структура identична Desktop, лише px інші).

Для агента-виконавця: всі точні значення (gap/padding/colors/font-size/border-radius) в розділах 4–5 — дослівні транскрипції з .fig, не округлювати до сітки 4/8px. Зображення копіювати з /Page-1/Desktop---4/assets/ (desktop) та відповідної /mobile/assets/ (mobile) — самі файли бінарно ідентичні між кадрами. SVG-обводки (Vector*.svg) копіювати як є.
